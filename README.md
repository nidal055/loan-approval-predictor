# 🍽️ Restaurant Review Sentiment Analysis

My second Machine Learning and Natural Language Processing (NLP) project.

This project classifies restaurant reviews as **Positive** or **Negative** using Natural Language Processing techniques and a **Multinomial Naive Bayes** classifier. The project demonstrates a complete NLP pipeline, from text preprocessing and feature extraction to model training, evaluation, and interactive sentiment prediction.

---

# 📌 Project Overview

Restaurants receive thousands of online reviews every day. Reading and categorizing these reviews manually is slow and inefficient.

This project automatically analyzes customer reviews and predicts whether the sentiment expressed in the review is **Positive** or **Negative**.

The model was trained using a real-world restaurant review dataset from Kaggle.

---

# 🚀 Features

- Load and preprocess restaurant review data
- Clean and normalize raw text
- Tokenization
- Stopword removal
- Lemmatization
- Convert text into numerical features using CountVectorizer
- Train a Multinomial Naive Bayes classifier
- Evaluate model performance
- Interactive sentence-level sentiment prediction

# 📚 Dataset

The project uses a public Restaurant Reviews dataset obtained from Kaggle.

Dataset characteristics:

- Real customer restaurant reviews
- Reviews labeled using review ratings
- Ratings above 3 → Positive
- Ratings below 3 → Negative
- Neutral ratings removed
- Missing values removed

Final dataset:

- **1,780 reviews**
- **1,280 Positive**
- **500 Negative**

---

# 🔍 NLP Pipeline

The project follows a complete Natural Language Processing workflow:

1. Load dataset
2. Clean text
3. Convert text to lowercase
4. Remove punctuation and special characters
5. Remove extra spaces
6. Tokenize text
7. Remove stopwords
8. Lemmatize words
9. Convert text into numerical vectors using CountVectorizer
10. Train the classifier
11. Evaluate the model
12. Predict sentiment for new reviews

---

# 🧠 Machine Learning Model

Classifier used:

**Multinomial Naive Bayes**

This algorithm is commonly used for text classification because it performs efficiently with word-frequency features generated by CountVectorizer.

---

# ✨ Text Preprocessing

The project applies several preprocessing techniques:

- Lowercase conversion
- Regular expression cleaning
- Tokenization
- Stopword removal
- Word lemmatization
- Whitespace normalization

Example:

Original review:

```
The Food was AMAZING!!!
```

Processed review:

```
the food was amazing
```

---

# 📈 Feature Extraction

Raw text cannot be processed directly by machine learning models.

The project uses **CountVectorizer** to convert every review into a vector of word frequencies.

Words such as:

- amazing
- delicious
- excellent

help predict positive sentiment, while words such as:

- bad
- rude
- terrible

help predict negative sentiment.

---

# 🎯 Model Training

The dataset is divided into:

- **80% Training**
- **20% Testing**

Stratified sampling preserves the original class distribution during training and evaluation.

---

# 📊 Model Evaluation

The model is evaluated using:

- Accuracy Score
- Actual vs Predicted Sentiment Comparison

A bar graph compares the true sentiment distribution with the predicted sentiment distribution to visualize model performance.

---

# 💬 Interactive Demo

The notebook includes an interactive demo where users can enter one or more sentences.

Example:

Input:

```
The food was amazing.
The waiter was rude.
```

Output:

```
Positive
Negative
```

Each sentence is processed independently using the trained model.

---

# ⚠ Limitations

Current limitations include:

- Binary classification only (Positive / Negative)
- No neutral sentiment class
- Limited understanding of sarcasm and context
- Relies on word-frequency features instead of semantic understanding

---

# 🔮 Future Improvements

Possible future enhancements include:

- Add Neutral sentiment classification
- Use TF-IDF instead of CountVectorizer
- Experiment with Logistic Regression and Support Vector Machines
- Implement transformer-based models such as BERT
- Support Arabic and Lebanese restaurant reviews
- Develop a Streamlit web application for easier interaction
